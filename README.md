## What is EDU?

**Eternal Dominion Universe** is an original worldbuilding and interactive fiction project. It chronicles the rise and fall of civilizations, legendary warriors, political factions, and extraordinary individuals across 500+ years of in-universe history (E1 to present day).

The story begins with humanity's great migration from Earth — through Andromeda, Leo, Sextans, and Triangulum galaxy clusters — finally settling in the **E16 binary star system**. What follows is centuries of war, culture, technological evolution, and survival.

The project lives as a living web app: an encyclopedia, a story reader, and a playable card game — all set in the same universe.

### 🃏 Card Game — `/card-game`
A **PvE deck-building battle game** built entirely from EDU universe lore.

**Deck Building**
- Pool of **64 cards** — one per wiki character
- Rarity tiers: C / R / SR
- Choose **5 cards** and set their battle order

**Battle System**
- All 5 characters deployed on the field from turn 1
- Each turn: select a character → choose an ability (Attack / Defense / Effect / Ultimate)
- Visual effects: particle bursts, slash animations, shield dome, screen shake, HP bar gradients
- 10 enemies across 4 difficulty tiers:

| Tier | Enemies |
|---|---|
| NORMAL | Stardust Dragon · Phantom Walker · Iron Golem · Venom Hydra |
| HARD | Frost Guardian · Flame Spirit King · Void Spider |
| BOSS | Fallen Angel · Void Reaper |
| FINAL | The Void King (HP: 100, ATK: 11) |

## Getting Started

**Prerequisites:** [Bun](https://bun.sh) ≥ 1.0

```bash
git clone https://github.com/gentaron/edu.git
cd edu
bun install
cp .env.example .env      # fill in DATABASE_URL etc.
bun prisma generate
bun prisma db push
bun dev                   # → http://localhost:3000
```

### Available Commands

```bash
bun dev              # development server (port 3000)
bun build            # production build
bun start            # serve production build
bun test             # run test suite (Vitest)
bun test:watch       # watch mode
bun run lint         # ESLint
bun run db:push      # sync Prisma schema → DB
bun run db:migrate   # run migrations
bun run db:reset     # reset database
```

## Roadmap

### Phase 1 — Foundation & Content (Near-term)

- [ ] **Image completion** — 193 wiki entries currently lack artwork. Target: all 64 character cards illustrated first, then organizations and locations.
- [ ] **Story expansion** — 8 characters have dedicated stories; 56 characters still need one. Starting with Tier 1 actives (Jen, Alpha Kane, Mina).
- [ ] **Wiki depth** — Expand descriptions for geography and technology entries. Many are stubs.
- [ ] **Mobile UX pass** — Card game battle screen is PC-first. Needs full mobile layout redesign.

### Phase 2 — Card Game Evolution (Mid-term)

- [ ] **Persistent player profile** — Save win/loss records, unlocked cards, cleared enemies locally via localStorage, then optionally sync to DB with NextAuth.
- [ ] **Roguelite mode** — Randomly generated enemy sequences with escalating difficulty. Permadeath run. Between battles: choose one of three random card upgrades.
- [ ] **Card upgrade system** — After clearing an enemy, evolve one card in your deck (e.g., C → R stat boost, new passive ability).
- [ ] **Animated card reveals** — Foil/holographic SR card flip animation on deck build screen.
- [ ] **Sound design** — Battle SFX (slash, shield, heal) and ambient tracks. Web Audio API or Howler.js.
- [ ] **Replay system** — Store battle turn logs in DB (`TurnLog` model already in schema). Allow sharing replays via URL.

### Phase 3 — NFT & Interoperability (Mid-term)

- [ ] **edunft repository** — Card metadata (abilities + lore text + image) stored as ERC-721 compatible JSON at [gentaron/edunft](https://github.com/gentaron/edunft).
- [ ] **IPFS image hosting** — Upload character artwork to Pinata. Replace GitHub raw URLs with `ipfs://` URIs for permanence.
- [ ] **Base blockchain minting** — Deploy ERC-1155 contract via Zora Protocol on Base mainnet. Each wiki card = one token ID. Mint price: 0 ETH (collectors pay only Base gas, ~$0.001).
- [ ] **Cross-project card import** — Any project can call `tokenURI(id)` or use Alchemy NFT API to retrieve card stats and use them in their own games.
- [ ] **Free claim page** — `/claim` — connect wallet, claim owned NFT cards into the card game deck builder.

### Phase 4 — Multiplayer (Long-term)

- [ ] **WebSocket infrastructure** — `examples/websocket/` already exists as a proof of concept. Promote to production service via `mini-services/`.
- [ ] **PvP async battles** — Challenge another player. Turns are submitted and resolved server-side. No real-time requirement — think chess-by-mail.
- [ ] **Real-time PvP** — Live 1v1 battle with WebSocket turn sync. Lobby system with matchmaking.
- [ ] **Guild / faction system** — Players align with EDU factions (AURALIS, Trinity Alliance, Alpha Venom, etc.). Faction leaderboard, weekly wars.
- [ ] **Tournament mode** — Bracket-style 8 or 16 player single-elimination. Automated scheduling, results tracking.

### Phase 5 — Platform Maturity (Long-term)

- [ ] **Full-text search** — Currently filter-only. Add fuzzy search across all 204 wiki entries + story text. Consider Fuse.js client-side or a search index API.
- [ ] **Internationalization** — All lore content is Japanese. Add English translations for wiki descriptions and story excerpts. `next-intl` already installed.
- [ ] **CMS integration** — Move wiki content from hardcoded TypeScript arrays to a headless CMS (Sanity or Contentlayer). Enable non-developer content editing.
- [ ] **Analytics** — Track which wiki pages, stories, and card combinations are most used. Privacy-respecting (Plausible or self-hosted Umami).
- [ ] **PWA** — Installable as a mobile app. Offline wiki access via service worker cache.
- [ ] **E2E tests** — Playwright coverage for the three critical paths: wiki navigation, story reading, and a complete card game run.

## Contributing

### Adding character artwork

Images are hosted at [gentaron/image](https://github.com/gentaron/image). Use PascalCase filenames matching the table below. Once uploaded, add the `image` field to the corresponding entry in `src/lib/wiki-data.ts`:

```ts
image: "https://raw.githubusercontent.com/gentaron/image/main/CharacterName.png",
```

193 entries are currently missing artwork — see the full filename reference in the [image tracking issue](#).

### Adding wiki entries

Edit `src/lib/wiki-data.ts`. Each entry follows this shape:

```ts
{
  id: "UniqueId",
  name: "表示名",
  nameEn: "English Name",          // optional
  category: "キャラクター",         // one of the 6 categories
  subCategory: "Gigapolis",         // optional
  description: "説明文...",
  era: "E300〜E350",                // optional
  affiliation: "Trinity Alliance",  // optional
  tier: "Tier 1",                   // optional
  image: "https://...",             // optional
  sourceLinks: [{ url: "...", label: "Story" }],  // optional
}
```
